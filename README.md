# Capítulo IV: Product Design

## 4.1. Style Guidelines

### 4.1.1. General Style Guidelines

**Branding:**
Para empezar, desarrollamos el logo del producto, con los colores iniciales que tendrá la aplicación. Asimismo, adaptamos el branding para que sea flexible al ser colocado en cualquier espacio de la interfaz del usuario.

![Imagen de Branding](./assets/branding.png)

**Typography:**
En cuanto a la fuente de texto, utilizaremos dos tipografías de Google Fonts: Mulish y Rubik. La primera fuente será de uso general para los párrafos de texto, artículos y anotaciones. Por otro lado, emplearemos Rubik para títulos, subtítulos, encabezados y botones. La escala de cada una de estas fuentes variará de acuerdo a las reglas de estilos aplicadas para los diseños responsivos encontrados en la Landing Page y la aplicación web.

![Imagen de Tipografía](./assets/typography.png)

**Spacing System:**
Para el sistema de espaciados, mantendremos una cantidad de 5 píxeles de distancia entre un elemento y otro distribuido en la interfaz.

![Imagen de Espaciados](./assets/spacing.png)

**Colors:**
Como última guía de estilos, escogimos colores que podrían agregar dinamismo, energía y pasión por la ruta a escoger en la aplicación.

![Imagen de Colores](./assets/colors.png)

### 4.1.2. Web Style Guidelines

Como startup, establecimos algunos aspectos de estilo con los que contará nuestra landing page y la aplicación web.

-**Interfaz intuitiva y avatar de guía inicial:**
En el caso de la aplicación web, cada vista contará con una ruta de uso intuitiva y un avatar personalizable que te ayuda con la primer interacción con una vista nueva, a modo de tutorial. Mientras que, en la landing page, se utilizarán convenciones y una jerarquía clásica propia de las landing page.

-**Contraste de colores condicionales:**
Como se mencionó en la guía de estilos generales, la interfaz podría cambiar de color, de modo que uno de los colores secundarios seleccionados puede reemplazar al color primario. Por ende, este color condicional combinará con el tema que escoja el usuario (tema claro u oscuro).

-**Diseño minimalista e impecable:**
La aplicación web tendrá un diseño minimalista, de modo que la información sea concisa y no resulte tedioso al leer.

## 4.2. Information Architecture

### 4.2.1. Organization Systems

Como equipo, decidimos que vamos a utilizar una estructura jerárquica (Hierarchical structure) para la landing page, con el fin de ofrecer nuestro producto y explicar sus beneficios desde las primeras capas que se visualizan al cargar la página por primera vez.

Por otro lado, la aplicación web contaría con una estructura secuencial (Sequential structure), debido a que se busca realizar una serie de pasos diariamente e ir registrando todo de forma constante. Además, las funcionalidades de paga también seguirán esta estructura, puesto que siguen una dinámica similar de completar un objetivo o buscan personalizar algo en base a las preferencias del usuario.

### 4.2.2. Labeling Systems

Dado que emplearemos un enfoque minimalista, la cantidad de texto en una vista sería reducida y, en su lugar, habrían más íconos o imágenes que ilustren lo que se busca realizar en dicha vista.

Para el uso de los botones, utilizaremos diferentes escalas de los colores detallados en nuestra guía de estilos.

### 4.2.4. Searching Systems

Los sistemas de búsqueda de nuestra aplicación realizarán consultas a APIs propias que irán creciendo con el paso del tiempo: una de comida y otra de ejercicios.

### 4.2.5. Navigation Systems

La forma de navegación para la landing page será mediante un header con botones que te redireccionan a cada sección de la página. Mientras que para la aplicación web, crearemos un sidebar con todas las vistas que necesita el usuario para disfrutar de las funcionalidades más importantes que ofrecemos.
